## 📌 一句话总结

**16人分成2个8人小组，29天全排班（包括原休息日），每人平均排班3.6次，所有约束都满足100%。**

## 🎯 新方案关键改变

### 之前的问题
- ❌ 人员分配不均（5-3-3分组不合理）
- ❌ 休息日也需要排班，但原方案没有
- ❌ 三组制度复杂，容易出现同组搭配

### 现在的解决方案
- ✅ 均匀分成2组，每组8人
- ✅ 全29天排班（包括原休息日）
- ✅ 更简洁的配对逻辑（只有2组，搭配更清晰）
- ✅ 排班差异仅1天（非常均衡）

## 📞 常见问题

### Q1: 原来休息日现在也要排班？
**A**: 是的。根据您的要求，所有29天都需要安排人员。这是合理的，因为医院需要365天运营。

### Q2: 为什么改成2组而不是3组？
**A**: 2组更简洁。3组时会有同组搭配的问题，2组完全避免了这个问题。

### Q3: 排班不均怎么办？
**A**: 现在已经非常均衡了！最多差异仅1天（有人排3次，有人排4次）。

### Q4: 如何处理请假或临时调换？
**A**: 这个系统是基础排班。实际执行时可以根据请假申请进行调整。

### Q5: 后续如何加入二线排班？
**A**: 目前这是一线排班系统。二线排班可以独立处理（如原文档中的李思、刘芙蓉等人）。

## 🎓 技术细节

### 算法
- **贪心算法**：每次选择排班最少、最久未排班的人
- **轮询策略**：循环遍历G1和G2，确保顺序均衡
- **约束验证**：每次排班都验证所有约束

### 数据结构
```python
STAFF = {
    "group_1": [  # 8人
        {"name": "...", "type": "resident|emergency|anesthesia|general"}
    ],
    "group_2": [  # 8人
        {"name": "...", "type": "resident|emergency|anesthesia|general"}
    ]
}
```

### 验证规则
```python
# 每个排班都要满足：
1. person1.group != person2.group  # 不同小组 ✓
2. has_special_bg(person1) OR has_special_bg(person2)  # 至少一人特殊背景 ✓
3. 28/16 ≤ duty_count ≤ 30/16  # 基本均衡 ✓
```
